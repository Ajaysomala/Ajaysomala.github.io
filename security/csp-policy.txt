# Content Security Policy Reference
# Somala Ajay Portfolio — ajaysomala.github.io
# ─────────────────────────────────────────────

## CURRENT ACTIVE POLICY (meta tag in index.html)

default-src 'self';
script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net https://cdnjs.cloudflare.com;
style-src 'self' 'unsafe-inline' https://fonts.googleapis.com;
font-src 'self' https://fonts.gstatic.com;
img-src 'self' data: https:;
connect-src 'self' https://api.emailjs.com https://*.emailjs.com https://api.github.com;
frame-ancestors 'none';
object-src 'none';
base-uri 'self';
upgrade-insecure-requests;

## WHY 'unsafe-inline' IS NEEDED
# GitHub Pages is a static host. Inline styles/scripts are required because:
# 1. CSS animations need inline style transitions
# 2. No build process to extract/hash inline code
# A nonce-based approach would be ideal but requires server-side rendering.

## THIRD-PARTY SOURCES WHITELISTED
# cdn.jsdelivr.net     — EmailJS SDK
# fonts.googleapis.com — Google Fonts CSS
# fonts.gstatic.com    — Google Fonts files
# api.emailjs.com      — Contact form email sending
# api.github.com       — GitHub stats (if used)
